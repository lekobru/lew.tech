Сборка и запуск
Установить зависимости: Visual Studio, Clang, WSDK (все в одном установщике).
Открыть Segregation.vcxproj через Visual Studio.
Отключить "Расширенный набор инструкций" в настройках проекта, если оные не поддерживаются.
Запустить сборку комбинацией Ctrl+Shift+B.
Запустить игру, открыть Segregation.cpl.
Предварительная конфигурация
Чтобы выровнять, насколько то возможно, снимок данных (углы, координаты, анимационные циклы) цели во время компенсаторики RTT в интерполяционной доле, что высчитывается при негативном расхождении между серверным и клиентским штампом времени обновления, и минимизировать сдвиг (ограниченный в ~200ms, если не учитывать sv_maxunlag - ответственный за компенсаторный предел RTT) от последнего штампа, тем самым продлевая окно попадания: cl_updaterate N;cl_interp_ratio 0;cl_interp 0;host_writeconfig, где N подразумевает частоту симуляции (в секунду) конкретного сервера.

Так как сервер отправляет лишь заключительные данные симуляции, а интерполяция строится из промежуточных данных, которые клиент упускает, как правило, то цель - исключить интерполяционное дробление (промежуточность), насколько это возможно, что в конечном счёте является основопологающей в нагромождении эффективностей искусственной задержки.

Для условных конфигураций утилизируется внутриигровая система програмированния (alias, bind) и исполнения программных файлов (exec). Например, переключение конфигурации между состоянием прыжка:

alias +conditional_jump "exec jump_condition;+jump"
alias -conditional_jump "exec unjump_condition;-jump"
bind space +conditional_jump
